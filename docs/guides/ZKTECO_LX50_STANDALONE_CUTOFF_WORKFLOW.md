# ZKTeco LX50 hybrid setup and cutoff export workflow

This guide is for the workflow where you will:

- use the **ZKTeco LX50 in a hybrid setup**
- connect the **Addbell Main Office** LX50 to a dedicated PC
- keep the other branch LX50 units on **manual USB export**
- add employees **directly on the device**
- download attendance logs to **USB every cutoff**
- review the file in **Excel**
- then move the data into this payroll system

Use this guide if:

- your **main office** will eventually use PC-assisted sync
- your **other 2 locations** will still export raw attendance logs manually every cutoff

---

## Recommended Addbell setup

Based on your intended setup, use this operating model:

| Location | Device mode | How records move | Recommended use |
|---|---|---|---|
| `Addbell Main Office` | LX50 connected to a PC | PC downloads or forwards punches into the payroll system | Main live source going forward |
| `Branch 1` | Standalone LX50 | HR exports attendance logs from device to USB every cutoff | Manual cutoff processing |
| `Branch 2` | Standalone LX50 | HR exports attendance logs from device to USB every cutoff | Manual cutoff processing |

### What "whatever the LX50 will generate" means

For the two manual locations, the payroll process should start from the **raw attendance file generated by the LX50**.

That raw file may be:

- an SSR attendance report
- an attendance log export
- a CSV-like text file
- an Excel-openable file

Do not require branch staff to retype time logs on paper first. The preferred source is always the **actual exported file from the device**.

### Best operating rule

Use one consistent rule across all locations:

- all employees are enrolled on the LX50 directly
- all employees use the same numeric PIN standard
- the LX50 `User ID / PIN` must always match `employees.employee_code`
- the main office device can push or sync to the app through the PC
- the other branches can stay manual until you are ready to automate them too

---

## What this system needs from the device

For the payroll system to recognize punches correctly, the most important rule is:

- the employee's **device User ID / PIN** on the LX50 must match `employees.employee_code` in this app

Do **not** use `company_id_no` for the device PIN unless it is exactly the same numeric value as `employee_code`.

Example:

- Employee record in this app:
  - `company_id_no`: `AX-10001`
  - `employee_code`: `25`
- LX50 device setup:
  - User ID / PIN: `25`

If the device PIN and `employee_code` do not match, imported logs will not map to the correct employee.

## Before you start

Prepare these first:

- the LX50 unit and power adaptor
- one USB flash drive, preferably **8GB to 16GB**, formatted as **FAT32**
- your employee list with:
  - employee name
  - numeric device PIN
  - optional department / notes
- access to this payroll system so you can verify each employee's `employee_code`

Recommended practice:

- assign a simple numeric PIN per employee such as `1`, `2`, `3`, `25`, `108`
- keep one employee = one PIN forever
- avoid reusing old PINs for resigned employees unless you are sure their old logs are already archived

## Part 1. First-time LX50 setup

Menu wording can vary a little by LX50 firmware, but the flow is usually very close to this:

- `Menu`
- `System`
- `Date Time` or `Date/Time`
- `Comm.` or `USB`
- `User Mng`
- `Data Mng`

### 1. Power on and set date/time

1. Plug in the LX50 and let it boot fully.
2. Open `Menu`.
3. Go to `System` > `Date Time` or similar.
4. Set:
   - correct date
   - correct time
   - correct time format
5. Save, then return to the main screen.

This matters because attendance logs are stored exactly as the device clock records them.

### 2. Set an admin on the device

1. Open `Menu` > `User Mng`.
2. Add or edit one account as the device administrator.
3. Give that admin either:
   - fingerprint admin access, or
   - password admin access
4. Save the admin record.

Do this so regular employees cannot edit users or clear logs.

### 3. Check USB and storage behavior

1. Insert your FAT32 USB flash drive.
2. Open `Menu` > `USB Mng`, `Data Mng`, or similar.
3. Confirm the device can detect the USB drive.
4. If the device shows options like:
   - `Download AttLog` 
   - `Download SSR`
   - `Download User`
   - `Upload User`
   note them down because you will use them every cutoff.

### 4. Decide your punch-state process

Your team needs a consistent rule for how employees punch:

- simplest: one punch for `IN`, one punch for `OUT`
- if the device asks for status, train employees to choose the correct state every time
- if your setup uses automatic state switching, test it before relying on it for payroll

If your export does not clearly show `IN` and `OUT`, payroll cleanup becomes manual later.

## Part 2. Add employees directly on the LX50

You can enroll employees from the device without using PC software.

### 1. Check the employee's time clock ID in this app

Before adding the employee to the LX50:

1. Open the employee record in this payroll system.
2. Find the employee's **time clock ID**, which is stored as `employee_code`.
3. Use that exact numeric value as the LX50 **User ID / PIN**.

### 2. Create the employee on the device

Typical flow:

1. On the LX50, open `Menu` > `User Mng`.
2. Choose `New User`, `Add User`, or similar.
3. Enter:
   - `User ID` or `Enroll No.`: use the app's `employee_code`
   - `Name`: employee name
   - `Role`: normal user
4. Save the basic user record.

### 3. Enroll the fingerprint

Typical flow:

1. With the user open, choose `Fingerprint`, `Enroll FP`, or similar.
2. Ask the employee to place the same finger on the scanner multiple times until the device confirms success.
3. If possible, enroll:
   - 1 primary finger
   - 1 backup finger
4. Save.

Best practice:

- enroll clean, dry fingers
- avoid enrolling a finger with cuts or unstable prints
- always test the fingerprint immediately after saving

### 4. Optional: add password as backup

If the LX50 allows password authentication:

1. Open the employee's user record.
2. Add a password only if your attendance policy allows it.
3. Save.

Use this carefully because password punches are easier to share than fingerprints.

### 5. Test the employee punch

After enrollment:

1. Have the employee punch once as `IN`.
2. If your process requires `OUT`, test that too.
3. Open `Record`, `Attendance`, `Query`, or `AttLog` on the device if available.
4. Confirm the employee's name/PIN and timestamp were recorded correctly.

Repeat this for each employee.

## Part 4. Download attendance logs every cutoff

This is the workflow for the two locations that will stay manual for now.

### 1. Prepare the USB drive

1. Use the same office USB if possible.
2. Make sure it is readable by the LX50.
3. If the device fails to read it, reformat the USB as `FAT32` and try again.

### 2. Download the logs from the LX50

Exact labels vary by firmware, but look for one of these menu paths:

- `Menu` > `USB Mng`
- `Menu` > `Data Mng`
- `Menu` > `SSR`

Then look for options such as:

- `Download AttLog`
- `Download Attendance`
- `Download SSR`
- `Download Report`

Recommended sequence:

1. Insert the USB flash drive.
2. Open the download/export menu.
3. Choose the attendance log export option.
4. Wait for the success message before removing the USB.

If the device offers both user export and attendance export, download:

- attendance logs every cutoff
- user list whenever you add or change many employees

### 3. Copy the file to your computer

1. Remove the USB safely from the LX50.
2. Insert it into your computer.
3. Copy the exported files into a folder such as:

```text
attendance-exports/
  2026-04-01_to_2026-04-15/
  2026-04-16_to_2026-04-30/
```

4. Do not edit the original raw export immediately. Keep one untouched backup copy first.

### 4. Open the file in Excel

Depending on the firmware/export type, the file may be:

- an Excel-readable file
- a CSV-like text file
- an SSR report file exported by the device

Open it in Excel and verify these fields as best as possible:

- user ID / PIN
- employee name, if included
- date
- time
- punch state, if included

If the export does not separate `IN` and `OUT`, you will need manual review before using it for payroll.

### 5. Keep the raw LX50 file as the source of truth

For the manual branches, the preferred source file is:

- the exact export generated by the LX50

Do this every cutoff:

1. Save the raw exported file in an archive folder.
2. Duplicate it into a working file for cleanup.
3. Use the working copy for Excel review and payroll preparation.

That way you always keep one untouched device-generated original.

## Part 5. How to move cutoff logs into this payroll system

### Current app status

This app currently supports:

- **live biometric/API import** through `POST /api/biometric/punch`
- **web-based AttLog upload/import** at `Time Entries` -> `Import AttLog`
- **manual time entry creation**
- **bulk manual time entry creation per employee** in `Time Entries`

The AttLog importer is designed for **raw LX50 attendance exports**. It lets HR:

- upload the exported file
- map the PIN/date/time/state columns
- preview the converted punches
- import them into `time_entries`

### What that means for your hybrid workflow

Right now you have **two practical tracks**:

#### Track A. Addbell Main Office: PC-connected device

For `Addbell Main Office`, the best setup is:

1. connect the LX50 to a dedicated PC
2. let the PC read or download the device records
3. send the punches into `POST /api/biometric/punch`
4. include the office location so main office punches are tagged correctly

This is the best long-term workflow because it removes manual cutoff encoding for the main office.

Reference:

- `docs/guides/ZKTECO_LX50_BIOMETRIC_SETUP.md`

#### Track B. Other 2 locations: raw export, then upload `AttLog`

Use this for the two other locations that will still operate in standalone mode.

Workflow:

1. Export attendance logs from the LX50 to USB.
2. Open and clean the file in Excel.
3. In the app, open `Time Entries` -> `Import AttLog`.
4. Upload the raw file and map the columns.
5. If the file already contains `IN` and `OUT`, use the file state.
6. If the file only contains timestamps, use the importer’s `Infer IN/OUT by punch order per employee/day` option carefully and review the preview first.
7. Import the punches into the system.

Important limitation:

- the importer depends on LX50 `PIN` matching `employees.employee_code`
- if the raw file has ambiguous punches, HR should review the preview before final import
- the old bulk entry flow is still **per employee** and is best for small manual corrections only

### Minimum data you should prepare from the raw LX50 export

Before encoding into the system, prepare one clean sheet with:

| employee_code | employee_name | date | time_in | time_out | notes |
|---|---|---|---|---|---|
| 25 | Juan Dela Cruz | 2026-04-15 | 08:01 | 17:09 | LX50 cutoff import |

That format is much easier to encode accurately than raw punch logs.

## Part 6. Suggested cutoff routine for HR

Use this every payroll cutoff:

1. For the two manual branches, export the latest attendance logs from the LX50 to USB.
2. For `Addbell Main Office`, verify the PC-connected device already downloaded or synced the latest punches.
3. Copy the raw branch files to your computer and archive them by cutoff period.
4. Open the branch exports in Excel.
5. Check for:
   - duplicate punches
   - missing `OUT`
   - wrong date/time
   - mismatched PINs
6. Convert each raw branch log into one clean daily sheet per employee.
7. Encode or import the cleaned daily attendance into this payroll workflow.
8. Review `Time Entries`, `Timesheet`, and payroll outputs before processing payslips.
9. Only after confirming backup copies exist should you clear old attendance logs from the device, if needed.

---

## Part 7. Important warnings

### Do not clear logs too early

Only clear attendance logs from the LX50 after:

- you already exported the cutoff data
- you copied the raw export to your computer
- you confirmed the file opens correctly
- payroll already has the needed records

### Keep device time accurate

If the LX50 time is wrong, payroll time will also be wrong.

Check the device clock:

- after brownouts or long power loss
- after daylight-saving or timezone-related changes
- at the start of each cutoff

### Keep one employee = one PIN

Do not recycle PINs casually. Historical exports depend on those IDs staying consistent.

## Part 8. Troubleshooting

### The USB drive is not detected

Try:

- reformatting the USB as `FAT32`
- using a smaller USB drive
- reconnecting the USB and reopening the menu

### The employee can enroll but cannot punch reliably

Try:

- re-enrolling the fingerprint
- enrolling a second finger
- cleaning the sensor
- testing with drier, firmer finger placement

### The exported file is confusing in Excel

That usually means the device gave you a raw attendance log instead of a cleaned payroll-ready sheet. Keep the raw file, then convert it into a clean daily sheet before encoding it into the app.

### The app cannot match the employee

Check whether:

- the LX50 `User ID / PIN` matches the employee's `employee_code`
- the employee is still active in the app
- the Excel sheet uses the correct numeric ID
